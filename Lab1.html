URL (Uniform Resource Locator) se usa oara especificar la direccion en la Web.

El protocolo especifica como se transporta la informacion. Por ejemplo, esta HTTP (HyperText Transfer Protocol).

Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es el conjunto de redes, mientras que la World Wide Web es un medio de acceso al Internet. La WWW funciona a traves de documentos de hipertexto(El hipertexto es una estructura no secuencial que permite crear, agregar, enlazar y compartir información de diversas fuentes por medio de enlaces asociativos y redes sociales)


¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET : usado para pedir data de una fuente especifica.

HEAD : es igual a GET, solo no retorna informacion.

POST : envia datos al servidor para crear o actualizar un recurso.

PUT : envia datos al servidor para crear o actualizar un recurso.

La diferencia entre PUT y POST es que PUT es idempotente, es decir, a partir de una aplicacion inicial, el resultado no cambia.

DELETE : elimina un recurso especifico.


¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Cuando se manda el formulario se usa el metodo GET. Cuando se responde el formulario, se usa el metodo POST.


¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No ocurrio ningun error, significa que "Se ha manejado correctamente una solicitud válida y, opcionalmente, se ha proporcionado una respuesta al usuario".


¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Si no fue ocasionado por el usuario (mal URL), es responsabilidad del desarrollador corregirlo.


¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si, porque un error 500 es un error interno de servido.


¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que un tag esta siendo reemplazado por uno nuevo o una mejor version. Un ejemplo es el tag de bold, con la version strong en HTML5.


¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Mejor manejo de errores y estructuracion de proyectos. Nuevos tags se estan creando y actualizando para optimizar la codificacion.
